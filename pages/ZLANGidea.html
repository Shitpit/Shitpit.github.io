Concept 01

Concept: planning on making a duolingo like website for learning the ZAZINs language in the website



problem one

how do i save the users data without using cookies so they can access their progress with any device without an account

soulution

I create a code that is a string of characters which determines where someone is up to in the language learning experience. ie. someone completes a lesson and gets given a code that unlocks a new lesson. for example a code could be: "LESSON=2&zazin%rand=7" the random number is so the person can't just guess the code and get to the next lesson. the LESSON= just shows what the player is up to and "&zazin" indicates the language

this also allows for a developer cheat code such as DEV%rand=2 to unlock all lessons for debugging purposes

Problems raised by this solution


Humans like me have a hard time remembering the codes so to shorten down the code we can develop a syntax that allows for (unmanitory) consiseness for example
you could type "LESSON=2&zazin%rand=2" but instead recommend using "Le2azp2 / LESSON(e = equals)2(a = @)z(p = percent) 2" so us humans can easily remember the code


On the topic of randomness

due to cross compatibility reasons such as wanting to use these codes in the future without advanced computer generation the random number is the lesson number converted to the pi equivilent 1=3 2=1 3=4 and then that gets turned into the a prime number for example lesson number one would be 5 (1 = 3 = 3rd prime number = 5) and lesson number 2 would be 2 (second digit of pi = 1, first prime number = 2) this is good for using this code or even making user generated language learning apps using this method
